ABC328C Consecutive 题解

Consecutive 题解

Description

给定一个 $N$，和长度为 $N$ 的字符串 $S$，以及 $Q$ 个询问，每个询问输入 $l,r$，求 $S_l,S_{l+1},S_{l+2},\dots,S_r$ 中有多少个相邻的字符相等。

Solution

一看就是经典的前缀和问题。

可以用一个 $sum$ 数组做前缀和，对于每个 $2 \le i \le N$：

  • 如果 $S_i=S_{i-1}$，$sum_i=sum_{i-1}+1$；
  • 否则，$sum_i=sum_{i-1}$。

ABC328D Take ABC 题解

Take ABC 题解

Description

给定一个只包含 A,B,C 的字符串 $S$，只要 $S$ 包含字串 ABC，就删除最左边的子串 ABC，输出最后的字符串。

Solution

这题类似于括号匹配，我们可以实现一个栈，然后遍历 $S$ 的每个字符，依次入栈，如果发现栈顶的 $3$ 个元素是 ABC，那么把 $3$ 个元素都弹出。

注意：RE 问题，弹出的时候要判断是否可以弹出。

ABC328E Modulo MST 题解

Modulo MST 题解

Description

有一个 $N$ 个节点 $M$ 条边的无向带权图，你需要选出一些边建造生成树。对于每一棵生成树，都有一个边权和 $T$。你需要求出所有 $T$ 对给定值 $K$ 求余的值中最小的是多少。

Solution

看到这么小的数据范围，你想到了什么？我想到了爆搜！

组合搜索加剪枝，搜出 $N - 1$ 条边，然后检查选出来的边是否可以构成一棵树，如果可以就更新答案。

ABC329C Count xxx 题解

ABC329C Count xxx 题解

Description

给定一个字符串，求其中有多少个所有字母均相同子串。

Solution

这题一看就知道是最长连号的改版，但是我太菜了，无脑写了个双指针。

所以此题提供两种思路。

双指针

双指针 $O(N)$ 查找子串，设两个指针分别为 $i,j$，开一个标记数组统计一下每个字符的出现次数，然后判断：

  • 如果当前的字符已经出现过了：
    • 如果出现次数小于现在的出现次数（$j-i+1$），那么就更新答案和标记数组。
    • 否则就什么也不干。
  • 否则，就记录次数，更新答案。

正解

开一个 $cnt$ 变量来记录出现次数，$ans$ 数组来统计出现次数，对于每个 $0 \le i \lt N$:

  • 如果 $i=0$ 或 $s_i \ne s_{i-1}$，那么 $cnt$ 变为 $1$；
  • 否则，$cnt$ 加 $1$
  • 然后每次更新 $ans_{s_{i}}$ 的数值即可。

ABC329D Election Quick Report 题解

ABC329D Election Quick Report 题解

Description

给定 $N,M$ 和 $M$ 个数，表示有 $N$ 个候选人和 $M$ 张选票，请你求出对于每个 $1 \le i \le N$：当投了前 $i$ 票时，谁的票数最多，如果有票数相同的输出编号最小的。

Solution

是不是感觉有点像 P7072 [CSP-J2020] 直播获奖？这题就是直播获奖的简化版！

于是，我们就可以开一个 multiset 记录每个候选人的票数和编号，然后输入的时候就删除再插入就可以了。为什么要这么做呢？因为 set 不支持修改元素！如果你这么做了，就会告诉你你修改的那一项是只读的。

注意：如果使用 pair，那么不能使用自带比较器。

ABC331E Set Meal 题解

ABC331E Set Meal 题解

Description

给定 $N$ 个主菜和 $M$ 个配菜，有 $L$ 个套餐不能配在一起，求套餐最大价格。

Solution

这题有一个奇怪的做法：利用题目的特点做。

具体怎么做呢？就是用 map 标记不能匹配的菜，对 $b$ 数组从大到小排序（注意 $b$ 数组要记录编号），然后枚举每个 $1 \le i \le N$，也就是枚举 $a$ 数组，再枚举每个 $1 \le j \le M$，也就是枚举 $b$ 数组，如果找到第一个没有被标记过的 $j$，那么退出，更新答案。

为什么能过呢？因为这个算法均摊时间复杂度为 $O(N+L)$！

大概结构：

  • 枚举 $i$：
    • 枚举 $j$：
      • 如果没有被标记过，退出；
      • 否则，继续。
    • 更新答案。

【ABC326C】题解

Peak 题解

Description

给定 $N,M$，还有 $N$ 个数 $a_1,a_2,\dots a_n$ 表示 $a_i$ 上有一个点，求出半开区间 $[x,x+M)$ 内的最大的点的个数。

Solution

既然是 $N$ 个点，我们就可以对 $a$ 排序，然后类似于双指针的方法来求解答案。

对于每个点 $i$：

  • 设一个 $j$，一开始等于 $i$；
    • 一直向后移动，直到 $a_{j + 1} - a_i \ge M$，就停止。
  • 然后就是更新答案：$j - i + 1$。

Time & Memory

时间：线性 $O(N)$；

空间：一个 $a$ 数组 $O(N)$。
